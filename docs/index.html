🏗️ Architecture

This app follows the MVVM (Model-View-ViewModel) design pattern, ensuring separation of concerns and maintainable code structure.

MVVM Pattern

📦 Model (CalculatorButton)
↓
🧠 ViewModel (CalculatorViewModel)
↓
📱 View (ContentView & CalculatorButtonView)

Key Components

  • CalculatorButton - Enum defining button types, colors, and properties
  • CalculatorViewModel - ObservableObject managing calculator state and business logic
  • ContentView - Main calculator interface with button grid
  • CalculatorButtonView - Reusable individual button component

📝 How It Works

The calculator maintains state through a ViewModel that tracks:

  • display - Current number or result shown to user
  • currentNumber - Working number being entered
  • previousNumber - First operand stored before operation
  • operation - Currently selected operation
  • isTypingNumber - Tracks if user is entering a number

Operation Flow

  1. User enters first number → stored in display
  2. User taps operation → number moved to previousNumber
  3. User enters second number → stored in display
  4. User taps equals → calculation performed and displayed
  5. Chained operations continue without breaking the flow
